# 🎯 Что реально нужно игроку во время игры

Игроку нужны три вещи:

1. Быстро бросать то, что он бросает чаще всего
2. Быстро считать то, что он считает чаще всего
3. Быстро управлять ресурсами

---

# 📌 1️⃣ Авто-броски из листа (самое логичное)

Раз у тебя уже есть карточка персонажа, то левую панель можно превратить в **панель быстрых действий персонажа**.

## Пример блока “Быстрые броски”

* 🎯 Атака (основное оружие)
* 🗡 Вторая атака (если есть)
* 💥 Урон
* 🛡 Спасброски (выпадающий список)
* 🧠 Проверки навыков (выпадающий список)

Кнопка делает:

```
1d20 + proficiency + ability mod
```

И результат летит в общий лог комнаты.

⚠ Важно:
Не делай автоматическое попадание.
Система просто пишет:

> Тарг — Атака длинным мечом: 17

Мастер сам решает, попал или нет.

# 📌 2️⃣ Авто-расчёт попадания (мягкий режим)

# Реализация: инструменты в левой колонке (игроки)

## Приоритет 1 — уже делаем
* **Быстрое изменение HP** — в левой панели (под кубиками) блок «Персонаж»: текущие HP, кнопки −1 / −5 и +1 / +5. Сохранение через API листа персонажа. Данные берём из `characterPreviews` (персонаж текущего игрока в комнате).

## Дальше (по тому же принципу)
* **Расчёт урона** — поле ввода числа урона или кнопка «Бросить урон» (из оружия в characterData.weapons: формула типа 2d6+3). Результат в лог; опционально кнопка «Применить урон» вычитает из HP и сохраняет.
* **Расчёт попадания** — кнопка «Атака» по выбранному оружию: 1d20 + модификатор атаки (из листа). В лог выводится только результат броска; мастер решает, попал ли удар. Опционально: при клике по NPC в бою показывать подсказку «Попадание (AC N)» без авто-применения.
* Всё это опирается на уже доступный лист: `characterData.weapons`, характеристики (СИЛ, ЛОВ и т.д.), proficiency из уровня — т.е. многое можно посчитать на клиенте без новых правил.
